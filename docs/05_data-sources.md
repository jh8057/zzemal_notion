# Daily Assistant — 데이터 소스 & 연동

## Notion 구조 (확인/정의 필요)

| DB/페이지     | 용도           | 가져올 필드          |
| ------------- | -------------- | -------------------- |
| 관심사/키워드 | 뉴스 수집 기준 | 키워드, 카테고리     |
| 스몰스탭      | 할일 목록      | 제목, 완료여부, 날짜 |
| 목표          | 장기 목표      | 제목, 기간, 진행률   |

> ⚠️ 실제 Notion DB 이름과 구조는 확인 후 업데이트 필요

## Google Calendar 연동

- 캘린더: 기본 캘린더 (+ Notion 연동된 캘린더)
- 가져올 것: 오늘 날짜 일정 (제목, 시간, 장소)

## 뉴스 수집 방법 (미결정)

| 방법                     | 장점             | 단점                 |
| ------------------------ | ---------------- | -------------------- |
| RSS 피드                 | 무료, 안정적     | 직접 소스 선정 필요  |
| Google Custom Search API | 키워드 검색 가능 | 하루 100건 무료 제한 |
| NewsAPI                  | 간단             | 무료 플랜 제한 있음  |

## LeetCode 문제 수집 (미결정)

| 방법 | 장점 | 단점 |
|---|---|---|
| LeetCode GraphQL API | 비공식이지만 널리 사용됨 | 공식 지원 아님 |
| 크롤링 | 자유도 높음 | 구조 변경 시 깨질 수 있음 |

## Gemini API 활용

- 입력: 오늘 일정 + 스몰스탭 내용
- 출력: 각 항목에 도움이 될 정보/팁/자료
- 모델: gemini-2.0-flash (무료 티어 충분)

## GitHub Actions 실행 환경

- Python 스크립트로 위 모든 데이터 수집
- API 키는 GitHub Secrets에서 환경변수로 주입
- 실행 환경: ubuntu-latest runner
